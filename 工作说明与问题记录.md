## 3. 你需要知道的基础概念

### 3.1 工作区

工作区就是我后续默认操作的文件夹。

你可以把它理解成这个项目的“主目录”。凡是和这个项目有关的文档、图片、数据、脚本，原则上都放在这里，避免桌面文件越来越乱。

### 3.2 Git 是什么

Git 是版本管理工具。

它解决的核心问题是：

- 记录文件每次修改
- 随时回看历史版本
- 在改错时可以回退
- 把本地项目同步到远程平台

你可以把 Git 理解为“项目历史记录系统”。

### 3.3 GitHub 是什么

GitHub 是托管 Git 仓库的网站。

它的作用主要是：

- 保存远程备份
- 在不同电脑之间同步
- 分享项目
- 管理 issue、PR、协作记录

简单说：

- `Git` 管本地历史
- `GitHub` 管远程仓库

### 3.4 gh 是什么

`gh` 是 GitHub 官方命令行工具。

它不是 Git 本身，而是 GitHub 的辅助操作工具。常见用途：

- 登录 GitHub
- 创建 GitHub 仓库
- 查看和创建 PR
- 管理 issue

这次安装 `gh` 的目的，是为了让我更直接地在本机上完成 GitHub 相关操作。

### 3.5 仓库是什么

仓库（repository，简称 repo）就是项目容器。

这里的仓库就是：

- 本地：`C:\Users\a1716\Desktop\attentional-blink`
- 远程：`OUYANGSHUHENG666/attentional-blink`

### 3.6 commit 是什么

`commit` 可以理解成一次“带说明的保存点”。

例如：

- 新增一篇整理稿
- 修正文献引用
- 调整文件结构

都可以形成一次 commit。

这样以后你就能知道：

- 改了什么
- 什么时候改的
- 为什么改

### 3.7 push 是什么

`push` 就是把本地提交上传到 GitHub 远程仓库。

流程通常是：

1. 先在本地修改文件
2. 用 Git 记录修改
3. 再 push 到 GitHub

### 3.8 branch 是什么

`branch` 是分支。

可以理解成一条独立修改线。

目前这个项目还比较简单，暂时只在主分支上工作。以后如果需要：

- 同时做不同版本
- 测试新结构
- 试验新写法

就可以单开分支，不影响主内容。
